
Ecstasy runner:

    Executes an Ecstasy module, compiling it first if necessary.

Usage:

    xec <options> <modulename>

Also supports any of:

    xec <options> <filename>
    xec <options> <filename>.x
    xec <options> <filename>.xtc


Options:
  -d | --deduce     Automatically deduce locations when possible
  --help            Display this help message
  -I | --inject     Specifies name/value pairs for injection; the format is "name1=value1,name2=value2"
  -J | --jit        Enable the JIT-to-Java back-end
  -L                Module path; a ":"-delimited list of file and/or directory names
  -M | --method     Method name; defaults to "run"
  --no-recompile    Disable automatic compilation
  -o                If compilation is necessary, the file or directory to write compiler output to
  -v | --verbose    Enable verbose logging and messages
  --version         Display the Ecstasy runtime version
  <_>               Module file name (.xtc) to execute
  ...               Arguments to pass to the method


Ecstasy compiler:

Converts ".x" files into a compiled ".xtc" Ecstasy module.

Usage:

    xcc <options> <filename>.x ...

Options:
  -d | --deduce     Automatically deduce locations when possible
  --help            Display this help message
  -L                Module path; a ":"-delimited list of file and/or directory names
  --nowarn          Suppress all warnings
  -o                File or directory to write output to
  --qualify         Use full module name for the output file name
  -r                Files and/or directories to read resources from
  --rebuild         Force rebuild
  --set-version     Specify the version to stamp onto the compiled module(s)
  --strict          Treat warnings as errors
  -v | --verbose    Enable verbose logging and messages
  --version         Display the Ecstasy runtime version
  ...               Source file name(s) and/or module location(s) to compile
